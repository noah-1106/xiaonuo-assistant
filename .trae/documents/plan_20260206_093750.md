# 工具调用统一化实施计划

## 目标
将所有工具调用（系统函数和模型工具）统一到`toolCall`格式中，减少大模型的理解压力，提高系统一致性。

## 实施步骤

### 1. 扩展executeToolCall方法
**文件**：`backend/src/services/taskService.js`

**修改内容**：
- 实现完整的`executeToolCall`方法
- 添加系统函数和模型工具的处理逻辑
- 支持所有现有的系统函数调用

### 2. 保持向后兼容性
**文件**：`backend/src/services/taskService.js`

**修改内容**：
- 保留`executeFunctionCall`方法
- 在`executeSubtask`中保持对`functionCall`的支持
- 确保现有任务仍能正常执行

### 3. 更新提示词和文档
**文件**：`backend/src/controllers/chatController.js`

**修改内容**：
- 更新工具定义中的提示词
- 强调使用统一的`toolCall`格式
- 提供详细的工具调用示例

## 技术实现

### 统一的toolCall格式
```json
"toolCall": {
  "name": "工具/函数名称",
  "arguments": "参数JSON字符串"
}
```

### 支持的工具类型
1. **系统函数**：
   - getRecentRecords
   - createRecord
   - getRecordList
   - updateRecord
   - deleteRecord
   - searchRecords
   - createTask
   - executeTask
   - getTaskList
   - getTask

2. **模型工具**：
   - web_search
   - image_process
   - video_process
   - file_process

### 核心修改代码

**executeToolCall方法**：
```javascript
async executeToolCall(toolCall) {
  const { name, arguments: argsStr } = toolCall;
  
  try {
    const args = JSON.parse(argsStr);
    
    // 系统函数处理
    switch (name) {
      case 'getRecentRecords':
        return await this.executeGetRecentRecords(args);
      case 'createRecord':
        return await this.executeCreateRecord(args);
      case 'getRecordList':
        return await this.executeGetRecordList(args);
      case 'updateRecord':
        return await this.executeUpdateRecord(args);
      case 'deleteRecord':
        return await this.executeDeleteRecord(args);
      case 'searchRecords':
        return await this.executeSearchRecords(args);
      case 'createTask':
        return await this.executeCreateTask(args);
      case 'executeTask':
        return await this.executeExecuteTask(args);
      case 'getTaskList':
        return await this.executeGetTaskList(args);
      case 'getTask':
        return await this.executeGetTask(args);
      
      // 模型工具处理
      case 'web_search':
      case 'Web_Search':
        return await this.executeWebSearch(args);
      case 'image_process':
      case 'Image_Process':
        return await this.executeImageProcess(args);
      case 'video_process':
      case 'Video_Process':
        return await this.executeVideoProcess(args);
      case 'file_process':
      case 'File_Process':
        return await this.executeFileProcess(args);
      
      default:
        return {
          status: 'completed',
          message: `工具调用执行成功: ${name}`,
          result: args
        };
    }
  } catch (error) {
    return {
      status: 'failed',
      message: `工具调用执行失败: ${error.message}`,
      error: error.message
    };
  }
}
```

### 系统函数实现
为每种系统函数添加具体的执行逻辑，调用相应的服务方法。

### 模型工具实现
为每种模型工具添加占位符实现，预留扩展空间。

## 预期效果

1. **大模型理解压力降低**：只需要学习一种调用格式
2. **系统一致性提高**：所有工具调用使用统一格式
3. **代码维护性提升**：减少代码复杂度
4. **向后兼容性保持**：现有任务仍能正常执行

## 风险评估

### 低风险
- **向后兼容性**：保留对`functionCall`的支持
- **系统稳定性**：修改集中在任务执行模块，影响范围有限

### 需关注
- **提示词更新**：确保大模型理解新的调用格式
- **错误处理**：添加完善的错误处理机制

此计划旨在通过统一工具调用格式，简化大模型的理解负担，同时保持系统的灵活性和兼容性。
