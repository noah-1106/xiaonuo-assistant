# createTask工具子任务结构优化计划

## 问题分析

**核心问题**：`createTask`工具的子任务结构不完整，缺少执行逻辑定义。

### 当前子任务结构
```json
{
  "title": "子任务标题",
  "description": "子任务描述"
}
```

### 问题影响
1. **任务无法执行**：AI生成的子任务没有执行逻辑，系统不知道如何执行
2. **工具调用失效**：无法通过子任务调用系统工具
3. **功能受限**：复杂任务无法通过多子任务协同完成

## 解决方案

### 修改createTask工具定义
在子任务的properties中添加执行逻辑字段：

1. **添加toolCall字段**：支持工具调用
2. **添加functionCall字段**：支持函数调用
3. **添加params字段**：支持普通子任务参数

### 具体修改内容

**文件**：`backend/src/controllers/chatController.js`

**修改点**：functions数组中createTask工具的定义

**修改内容**：
- 在subtasks.items.properties中添加以下字段：
  - `toolCall`：工具调用配置
  - `functionCall`：函数调用配置  
  - `params`：子任务参数

### 预期效果

**修改后子任务结构**：
```json
{
  "title": "子任务标题",
  "description": "子任务描述",
  "toolCall": {
    "name": "工具名称",
    "arguments": "工具参数JSON字符串"
  }
  // 或
  "functionCall": {
    "name": "函数名称",
    "arguments": "函数参数JSON字符串"
  }
  // 或
  "params": {
    "key": "value"
  }
}
```

### 验证计划

1. **修改工具定义**：更新createTask工具的子任务结构
2. **重启服务**：让修改生效
3. **测试任务创建**：发送测试请求创建包含执行逻辑的任务
4. **验证执行**：测试任务执行功能
5. **检查日志**：确认工具调用流程正常工作

## 技术要点

- **向后兼容**：保持现有字段不变，只添加新字段
- **清晰定义**：为新字段添加详细的描述和示例
- **一致性**：确保与系统其他部分的工具调用格式保持一致
- **可扩展性**：设计灵活的结构，支持未来添加新的执行方式

## 执行步骤

1. **修改工具定义**：更新createTask工具的子任务结构定义
2. **重启后端服务**：应用修改
3. **测试验证**：创建包含执行逻辑的任务并验证执行
4. **优化调整**：根据测试结果进行必要的调整
