# 步进式子任务执行系统实现计划

## 1. 系统现状分析

### 当前任务执行流程
- 任务创建后一次性执行所有子任务
- 缺乏子任务级别的状态通知
- AI调用可能导致整个流程超时
- 前端无法实时了解执行进度

### 核心问题
- 单次AI调用上下文过长
- 缺乏细粒度的错误处理
- 用户体验较差，无法感知执行进度

### 2.2 关键功能实现

#### 步进式子任务执行
```javascript
async executeSubtasks(taskId, userId) {
  const task = await Task.findById(taskId);
  const subtasks = task.subtasks || [];
  
  for (let i = 0; i < subtasks.length; i++) {
    const subtask = subtasks[i];
    
    // 1. 发送子任务开始通知
    await this.notifySubtaskStart(userId, taskId, i, subtask);
    
    // 2. 执行子任务
    let subtaskResult;
    try {
      if (subtask.functionName) {
        // 系统可直接执行的子任务
        subtaskResult = await this.executeSystemSubtask(subtask);
      } else if (subtask.requiresAI) {
        // 需要AI参与的子任务
        subtaskResult = await this.executeAISubtask(userId, taskId, subtask);
      } else {
        // 描述性子任务，需要AI生成执行逻辑
        subtaskResult = await this.generateAndExecuteSubtask(userId, taskId, subtask);
      }
      
      // 3. 发送子任务完成通知
      await this.notifySubtaskComplete(userId, taskId, i, subtask, subtaskResult);
      
      // 4. 更新任务进度
      task.progress = Math.round((i + 1) / subtasks.length * 100);
      await task.save();
      
    } catch (error) {
      // 5. 发送子任务错误通知
      await this.notifySubtaskError(userId, taskId, i, subtask, error.message);
      
      // 6. 决定是否继续执行
      if (task.stopOnError) {
        throw error;
      }
    }
  }
  
  // 7. 任务执行完成
  task.status = 'completed';
  await task.save();
  await this.notifyTaskComplete(userId, task);
  
  return task;
}
```

#### 实时状态通知
```javascript
// 子任务开始通知
async notifySubtaskStart(userId, taskId, subtaskIndex, subtask) {
  await this.io.to(userId).emit('subtask_started', {
    taskId,
    subtaskIndex,
    subtask: {
      title: subtask.title,
      description: subtask.description
    },
    timestamp: new Date()
  });
}

// 子任务完成通知
async notifySubtaskComplete(userId, taskId, subtaskIndex, subtask, result) {
  await this.io.to(userId).emit('subtask_completed', {
    taskId,
    subtaskIndex,
    subtask: {
      title: subtask.title,
      description: subtask.description
    },
    result,
    timestamp: new Date()
  });
}
```

// 处理子任务开始通知
socket.on('subtask_started', (data) => {
  const task = tasks.find(t => t.id === data.taskId);
  if (task) {
    task.status = 'running';
    task.currentSubtask = data.subtask;
    task.subtaskIndex = data.subtaskIndex;
    updateTask(task);
    
    // 添加系统消息
    addSystemMessage({
      id: generateId(),
      content: `开始执行子任务: ${data.subtask.title}`,
      timestamp: new Date()
    });
  }
});

// 处理子任务完成通知
socket.on('subtask_completed', (data) => {
  const task = tasks.find(t => t.id === data.taskId);
  if (task) {
    task.progress = Math.round(((data.subtaskIndex + 1) / (task.subtasks?.length || 1)) * 100);
    updateTask(task);
    
    // 添加系统消息
    addSystemMessage({
      id: generateId(),
      content: `子任务完成: ${data.subtask.title}`,
      timestamp: new Date()
    });
  }
});

## 3. 技术优势

### 性能优化
- **降低响应要求**：每次只处理一个子任务，响应时间更可控
- **减轻上下文压力**：单个子任务的上下文长度大大减少
- **减少超时风险**：单个子任务执行时间短，不易触发超时

### 用户体验提升
- **实时反馈**：用户可以看到每个子任务的执行状态
- **进度可视化**：通过进度百分比直观了解任务执行情况
- **错误处理透明**：子任务错误可以及时通知用户，不影响整个任务

### 系统可靠性
- **容错能力**：单个子任务失败可以选择继续执行其他子任务
- **可恢复性**：任务执行中断后可以从失败的子任务开始继续执行
- **可扩展性**：支持添加新的子任务类型和执行逻辑
