# Gitee代码和版本管理详细规划

## 一、准备工作

### 1. 完善.gitignore文件

* **目标**：确保不提交不必要的文件到仓库

* **步骤**：

  1. 编辑`.gitignore`文件
  2. 添加以下内容：

     ```
     # 依赖目录
     node_modules/
     pnpm-lock.yaml

     # 环境变量文件
     .env
     .env.local
     .env.*.local

     # 日志文件
     logs
     *.log

     # 编译输出
     dist/
     build/

     # IDE配置
     .vscode/
     .idea/
     *.swp
     *.swo

     # 系统文件
     .DS_Store
     Thumbs.db

     # 证书和密钥
     *.pem
     *.key

     # 临时文件
     .tmp/
     temp/

     # 部署文件
     nginx.conf
     xiaonuo.top_nginx/
     ```

### 2. 检查当前Git状态

* **目标**：了解当前仓库状态

* **步骤**：

  1. 运行`git status`命令
  2. 确认所有未跟踪文件
  3. 检查是否有需要忽略的文件

## 二、本地仓库初始化

### 1. 初始化Git仓库（如果尚未初始化）

* **目标**：创建本地Git仓库

* **步骤**：

  1. 运行`git init`命令
  2. 确认仓库创建成功

### 2. 添加文件到暂存区

* **目标**：准备提交文件

* **步骤**：

  1. 运行`git add .`添加所有文件
  2. 运行`git status`确认文件已添加

### 3. 创建初始提交

* **目标**：创建第一个版本

* **步骤**：

  1. 运行`git commit -m "初始化仓库：添加小诺智能助理项目代码"`
  2. 确认提交成功

## 三、Gitee仓库创建

### 1. 登录Gitee账号

* **步骤**：

  1. 访问Gitee官网：<https://gitee.com/>
  2. 登录或注册账号

### 2. 创建新仓库

* **步骤**：

  1. 点击右上角"+"号，选择"新建仓库"
  2. 填写仓库信息：

     * 仓库名称：xiaonuo-assistant

     * 仓库介绍：小诺智能助理 - 基于AI的智能助手系统

     * 选择公开或私有

     * 不勾选"使用README.md初始化仓库"
  3. 点击"创建仓库"

### 3. 复制仓库地址

* **步骤**：

  1. 在仓库页面，复制HTTPS或SSH地址
  2. 例如：`https://gitee.com/your-username/xiaonuo-assistant.git`

## 四、远程仓库配置

### 1. 添加Gitee作为远程仓库

* **目标**：关联本地仓库和Gitee仓库

* **步骤**：

  1. 运行`git remote add origin https://gitee.com/your-username/xiaonuo-assistant.git`
  2. 运行`git remote -v`确认远程仓库添加成功

### 2. 配置用户信息（如果尚未配置）

* **目标**：确保提交时包含正确的作者信息

* **步骤**：

  1. 运行`git config --global user.name "Your Name"`
  2. 运行`git config --global user.email "your.email@example.com"`

## 五、代码推送

### 1. 推送代码到Gitee

* **目标**：将本地代码上传到Gitee

* **步骤**：

  1. 运行`git push -u origin main`
  2. 输入Gitee账号密码或SSH密钥验证
  3. 等待推送完成

### 2. 验证推送结果

* **目标**：确认代码成功上传

* **步骤**：

  1. 刷新Gitee仓库页面
  2. 检查文件是否显示
  3. 确认提交记录存在

## 六、版本管理策略

### 1. 分支管理

* **目标**：建立清晰的分支结构

* **分支策略**：

  * `main`分支：稳定版本，用于生产环境

  * `develop`分支：开发分支，集成所有功能

  * `feature/*`分支：特性分支，用于开发新功能

  * `fix/*`分支：修复分支，用于修复bug

### 2. 创建分支

* **步骤**：

  1. 创建开发分支：`git checkout -b develop`
  2. 推送开发分支：`git push origin develop`
  3. 创建特性分支：`git checkout -b feature/login-page`
  4. 推送特性分支：`git push origin feature/login-page`

### 3. 分支合并

* **步骤**：

  1. 切换到目标分支：`git checkout develop`
  2. 合并特性分支：`git merge feature/login-page`
  3. 推送合并结果：`git push origin develop`
  4. 删除已完成的特性分支：`git branch -d feature/login-page`
  5. 删除远程特性分支：`git push origin --delete feature/login-page`

## 七、提交规范

### 1. 语义化提交格式

* **格式**：`<type>(<scope>): <description>`

* **示例**：`feat(login): 添加验证码登录功能`

### 2. 提交类型

* `feat`：新功能

* `fix`：修复bug

* `docs`：文档更新

* `style`：代码风格修改

* `refactor`：代码重构

* `test`：测试相关

* `chore`：构建过程或辅助工具的变动

### 3. 提交示例

* 功能开发：`git commit -m "feat(chat): 实现AI聊天功能"`

* Bug修复：`git commit -m "fix(auth): 修复验证码发送失败的问题"`

* 文档更新：`git commit -m "docs(readme): 更新项目说明文档"`

## 八、常见操作

### 1. 拉取最新代码

* **步骤**：

  1. 运行`git pull origin main`

### 2. 查看提交历史

* **步骤**：

  1. 运行`git log --oneline`查看简洁历史
  2. 运行`git log`查看详细历史

### 3. 撤销修改

* **未添加到暂存区**：`git checkout -- <file>`

* **已添加到暂存区**：`git reset HEAD <file>`

* **已提交**：`git revert <commit-hash>`

### 4. 标签管理

* **创建标签**：`git tag v1.0.0`

* **推送标签**：`git push origin v1.0.0`

* **查看标签**：`git tag`

## 九、常见问题和解决方案

### 1. 推送失败

* **原因**：可能是权限问题或网络问题

* **解决方案**：

  * 检查Gitee账号权限

  * 检查网络连接

  * 确认远程仓库地址正确

### 2. 冲突解决

* **原因**：多人修改同一文件导致冲突

* **解决方案**：

  1. 查看冲突文件
  2. 手动编辑解决冲突
  3. 运行`git add <file>`标记冲突已解决
  4. 运行`git commit`完成合并

### 3. 忘记提交文件

* **解决方案**：

  1. 运行`git add <file>`
  2. 运行`git commit --amend`修改上次提交

## 十、后续维护

* **建议**：每天结束工作时推送代码

* **步骤**：

  1. `git add .`
  2. `git commit -m "日常更新：添加功能和修复bug"`
  3. `git push origin <branch>`

### 2. 定期拉取代码

* **建议**：每天开始工作时拉取代码

* **步骤**：

  1. `git pull origin <branch>`

### 3. 代码审查

* **建议**：使用Gitee的Pull Request功能

* **步骤**：

  1. 在Gitee上创建Pull Request
  2. 邀请团队成员审查代码
  3. 合并代码到主分支
