# 系统环境设计与访问权限配置指南

本文档提供了小诺智能助理系统的环境设计和访问权限配置的详细步骤，指导您完成火山引擎资源的创建、配置和权限设置，确保系统能够安全、稳定地运行。

## 一、环境设计概述

### 1. 系统架构

```
┌───────────────────────────────────────┐
│               公网                     │
├───────────────────────────────────────┤
│ ┌──────────┐  ┌──────────┐  ┌────────┐ │
│ │   CDN    │  │  CLB     │  │  域名  │ │
│ └──────────┘  └──────────┘  └────────┘ │
├───────────────────────────────────────┤
│ ┌──────────────────────────────────┐  │
│ │           私有网络 VPC           │  │
│ ├──────────┐ ┌──────────┐ ┌────────┐ │
│ │ 前端服务 │ │ 后端服务 │ │ MongoDB│ │
│ └──────────┘ └──────────┘ └────────┘ │
│ ┌──────────┐ ┌──────────┐ ┌────────┐ │
│ │   TOS    │ │  监控    │ │  日志  │ │
│ └──────────┘ └──────────┘ └────────┘ │
│ └──────────────────────────────────┘  │
└───────────────────────────────────────┘
```

### 2. 资源规划

| 资源类型 | 规格 | 数量 | 用途 |
|---------|------|------|------|
| 云服务器 ECS | 4核4G | 1 | 部署前端和后端服务 |
| 数据库 MongoDB | 2核4G | 1 | 数据存储 |
| 对象存储 TOS | 标准存储 | 1 | 文件存储 |
| 内容分发网络 CDN | 中国大陆 | 1 | 静态资源加速 |
| 负载均衡 CLB | 基础型 | 1 | 流量分发 |
| 私有网络 VPC | 标准 | 1 | 网络隔离 |
| 安全组 | 标准 | 2 | 网络访问控制 |

## 三、VPC网络配置

### 步骤1：创建私有网络

1. 登录火山引擎控制台
2. 在左侧导航栏选择「网络」→「私有网络 VPC」
3. 点击「创建私有网络」按钮
4. 配置参数：
   - 名称：`xiaonuo-vpc`
   - 地域：选择您所在的地域（建议选择靠近用户的地域）
   - IPv4网段：`192.168.0.0/16`  //最大到16，已按照16进行创建
   - 描述：小诺智能助理私有网络
5. 点击「确定」按钮

### 步骤2：创建子网

1. 在私有网络详情页面，点击「子网」标签页
2. 点击「创建子网」按钮
3. 配置参数：
   - 名称：`xiaonuo-subnet`
   - 可用区：选择默认可用区
   - IPv4网段：`192.168.1.0/24` //最大到16，已按照16进行创建
   - 描述：小诺智能助理子网
4. 点击「确定」按钮

### 步骤3：创建安全组

1. 在左侧导航栏选择「网络」→「安全组」
2. 点击「创建安全组」按钮
3. 配置参数：
   - 名称：`xiaonuo-server-sg`
   - 地域：与VPC相同
   - VPC：选择 `xiaonuo-vpc`
   - 描述：服务器安全组
4. 点击「确定」按钮

5. 重复上述步骤，创建第二个安全组：
   - 名称：`xiaonuo-db-sg`
   - 描述：数据库安全组

## 四、云服务器配置

### 步骤1：创建云服务器

1. 在左侧导航栏选择「计算」→「云服务器 ECS」
2. 点击「创建实例」按钮
3. 配置参数：
   - 计费方式：按量计费
   - 地域：与VPC相同
   - 可用区：与子网相同
   - 实例规格：选择「4核4G」（建议：ecs.c6.xlarge）
   - 镜像：选择「CentOS 7.9」  //系统为veLinux 2.0 CentOS Compatible 64 bit
   - 系统盘：40GB SSD云盘
   - 网络：
     - 选择私有网络：`xiaonuo-vpc`
     - 选择子网：`xiaonuo-subnet`
     - 公网IP：勾选「分配公网IPv4地址」
     - 带宽：2Mbps
   - 安全组：选择 `xiaonuo-server-sg`
   - 登录方式：选择「密钥对」
     - 点击「新建密钥对」  //已有
     - 名称：`xiaonuo-key`  //你先用的就是
     - 点击「确定」并下载密钥文件  //你现在用的就是
   - 实例名称：`xiaonuo-server`  //目前使用的是xiaonuo1
   - 描述：小诺智能助理服务器
4. 点击「立即购买」按钮
5. 确认订单并完成支付

### 步骤2：配置安全组规则

1. 在云服务器列表页面，点击实例名称进入详情页
2. 点击「安全组」标签页
3. 点击「配置规则」按钮
4. 添加以下入站规则：
   - 规则1：
     - 端口范围：80/80
     - 授权对象：0.0.0.0/0
     - 描述：HTTP访问
   - 规则2：
     - 端口范围：443/443
     - 授权对象：0.0.0.0/0
     - 描述：HTTPS访问
   - 规则3：
     - 端口范围：3001/3001
     - 授权对象：0.0.0.0/0
     - 描述：后端服务访问
   - 规则4：
     - 端口范围：22/22
     - 授权对象：0.0.0.0/0
     - 描述：SSH访问
5. 点击「确定」按钮

### 步骤3：连接服务器

1. 使用下载的密钥文件连接服务器：
   ```bash
   # 在本地终端执行
   chmod 600 /path/to/xiaonuo-key.pem
   ssh -i /path/to/xiaonuo-key.pem root@服务器公网IP
   ```

2. 验证连接成功：
   - 成功登录服务器后，会看到CentOS的欢迎信息

## 七、CDN配置

### 步骤1：添加加速域名

1. 在左侧导航栏选择「CDN与智能边缘」→「内容分发网络 CDN」
2. 点击「添加加速域名」按钮
3. 配置参数：
   - 加速域名：输入您的域名（例如：static.xiaonuo.top）
   - 业务类型：静态网站加速
   - 源站配置：
     - 源站类型：IP
     - 源站地址：输入云服务器的公网IP
     - 端口：80
   - 回源Host：输入加速域名
   - 缓存配置：使用默认配置
   - HTTPS配置：
     - 开启HTTPS：是
     - 证书类型：选择「免费证书」
     - 点击「申请免费证书」
     - 输入域名并完成DNS验证
4. 点击「确定」按钮

### 步骤2：配置DNS解析

1. 登录您的域名注册商后台
2. 找到域名解析设置
3. 添加以下解析记录：
   - 记录类型：A
   - 主机记录：static
   - 记录值：输入CDN分配的CNAME地址
   - TTL：10分钟
4. 保存解析设置

     ```json
             "clb:*",
     ```

## 十一、系统环境配置

### 步骤1：登录服务器

1. 使用密钥文件登录云服务器：
   ```bash
   ssh -i /path/to/xiaonuo-key.pem root@服务器公网IP
   ```

### 步骤2：安装依赖

1. 更新系统包：
   ```bash
   yum update -y
   ```

2. 安装必要的依赖：
   ```bash
   yum install -y gcc-c++ make curl wget git nginx
   ```

3. 安装 Node.js 18.x：
   ```bash
   curl -fsSL https://deb.nodesource.com/setup_18.x | bash -
   yum install -y nodejs
   ```

4. 验证安装：
   ```bash
   node -v
   npm -v
   ```

### 步骤3：安装 MongoDB 客户端

1. 创建 MongoDB 仓库文件：
   ```bash
   vi /etc/yum.repos.d/mongodb-org-4.4.repo
   ```

2. 添加以下内容：
   ```
   [mongodb-org-4.4]
   name=MongoDB Repository
   baseurl=https://repo.mongodb.org/yum/redhat/$releasever/mongodb-org/4.4/x86_64/
   gpgcheck=1
   enabled=1
   gpgkey=https://www.mongodb.org/static/pgp/server-4.4.asc
   ```

3. 安装 MongoDB 客户端：
   ```bash
   yum install -y mongodb-org-shell
   ```

4. 验证安装：
   ```bash
   mongo --version
   ```

### 步骤4：配置 Nginx

1. 备份默认配置：
   ```bash
   mv /etc/nginx/nginx.conf /etc/nginx/nginx.conf.bak
   ```

2. 创建新的 Nginx 配置：
   ```bash
   vi /etc/nginx/nginx.conf
   ```

3. 添加以下内容：
   ```nginx
   user nginx;
   worker_processes auto;
   error_log /var/log/nginx/error.log;
   pid /run/nginx.pid;

   events {
       worker_connections 1024;
   }

   http {
       log_format  main  '$remote_addr - $remote_user [$time_local] "$request" '
                         '$status $body_bytes_sent "$http_referer" '
                         '"$http_user_agent" "$http_x_forwarded_for"';

       access_log  /var/log/nginx/access.log  main;

       sendfile            on;
       tcp_nopush          on;
       tcp_nodelay         on;
       keepalive_timeout   65;
       types_hash_max_size 2048;

       include             /etc/nginx/mime.types;
       default_type        application/octet-stream;

       # 前端应用配置
       server {
           listen       80;
           server_name  xiaonuo.top www.xiaonuo.top;

           location / {
               root   /var/www/xiaonuo;
               index  index.html;
               try_files $uri $uri/ /index.html;
           }

           # API 代理
           location /api {
               proxy_pass http://localhost:3001;
               proxy_http_version 1.1;
               proxy_set_header Upgrade $http_upgrade;
               proxy_set_header Connection 'upgrade';
               proxy_set_header Host $host;
               proxy_cache_bypass $http_upgrade;
           }

           error_page 404 /404.html;
               location = /40x.html {
           }

           error_page 500 502 503 504 /50x.html;
               location = /50x.html {
           }
       }
   }
   ```

4. 创建前端应用目录：
   ```bash
   mkdir -p /var/www/xiaonuo
   ```

5. 重启 Nginx 服务：
   ```bash
   systemctl restart nginx
   systemctl enable nginx
   ```

## 十二、部署脚本配置

### 步骤1：上传部署脚本

1. 在本地创建部署脚本目录：

## 十五、监控与维护

### 步骤1：配置云监控

1. 在火山引擎控制台，选择「监控与运维」→「云监控」
2. 点击「创建告警规则」按钮
3. 配置以下告警规则：
   - 服务器 CPU 使用率 > 80% 持续5分钟
   - 服务器内存使用率 > 85% 持续5分钟
   - 服务器磁盘使用率 > 90% 持续10分钟
   - MongoDB 连接数 > 100 持续5分钟
   - CDN 命中率 < 70% 持续10分钟
4. 设置告警通知：
   - 选择通知方式：邮件、短信
   - 输入通知接收人信息

1. 在火山引擎控制台，选择「日志服务」→「日志项目」
2. 点击「创建日志项目」按钮
3. 名称：`xiaonuo-log`
4. 地域：与VPC相同
5. 点击「确定」按钮

6. 在日志项目详情页，点击「创建日志主题」按钮
7. 名称：`xiaonuo-server-log`
8. 点击「确定」按钮

9. 配置日志采集：
   - 选择「服务器日志」
   - 选择云服务器 `xiaonuo-server`
   - 配置日志路径：`/var/log/nginx/access.log`
   - 点击「确定」按钮

## 十七、故障排查

### 常见问题及解决方案

| 问题 | 可能原因 | 解决方案 |
|------|---------|----------|
| 服务器无法连接 | 安全组规则未配置 | 检查安全组入站规则，确保22端口开放 |
| 应用无法访问 | Nginx配置错误 | 检查Nginx配置文件和服务状态 |
| 数据库连接失败 | MongoDB连接信息错误 | 检查数据库连接字符串和密码 |
| TOS上传失败 | 密钥配置错误 | 检查TOS访问密钥和存储桶权限 |
| CDN加速无效 | DNS解析未生效 | 检查DNS解析记录和TTL设置 |
| 部署脚本执行失败 | 权限不足 | 检查服务器密钥权限和路径配置 |
