## 报告安全漏洞

小诺智能助理团队非常重视安全问题。我们感谢安全研究人员和社区成员负责任地披露安全漏洞。

### 如何报告

**请不要在公共 Issue 中报告安全漏洞。**

如果你发现了安全漏洞，请通过以下方式之一私下报告：

- 发送邮件至：[noah-tan@live.com](mailto:noah-tan@live.com)
- 使用 GitHub 的 [Private Vulnerability Reporting](https://github.com/yourusername/xiaonuo/security/advisories/new) 功能

### 报告内容

请在你的报告中包含以下信息：

1. **漏洞描述**：清晰简洁地描述漏洞
2. **影响范围**：受影响的版本和组件
3. **复现步骤**：详细的复现步骤
4. **影响评估**：潜在的安全影响
5. **修复建议**（可选）：如果你有任何修复建议
6. **联系方式**：你的联系方式以便我们跟进

### 响应流程

1. **确认**：我们将在 48 小时内确认收到你的报告
2. **评估**：我们将评估漏洞的严重性和影响
3. **修复**：我们将开发修复方案
4. **发布**：修复完成后，我们将发布安全更新
5. **致谢**：我们将在安全公告中感谢你的贡献（如果你愿意）

## 安全最佳实践

### 部署安全

1. **使用 HTTPS**
   - 始终在生产环境使用 HTTPS
   - 配置 HSTS 头
   - 使用现代 TLS 版本（TLS 1.2+）

2. **环境变量保护**
   - 永远不要提交 `.env` 文件到版本控制
   - 使用强密码和密钥
   - 定期轮换 API 密钥

3. **数据库安全**
   - 使用强密码
   - 限制数据库访问
   - 启用身份验证
   - 定期备份

4. **文件上传安全**
   - 验证文件类型
   - 限制文件大小
   - 使用安全的存储服务

### 代码安全

1. **依赖管理**
   ```bash
   # 定期运行安全审计
   pnpm audit
   
   # 更新依赖
   pnpm update
   ```

2. **输入验证**
   - 始终验证用户输入
   - 使用参数化查询防止 SQL 注入
   - 转义输出防止 XSS

3. **认证和授权**
   - 使用强 JWT 密钥
   - 实现适当的访问控制
   - 定期清理过期会话

### 监控和日志

1. **日志记录**
   - 记录安全相关事件
   - 定期审查日志
   - 保护日志文件

2. **监控**
   - 设置异常检测
   - 监控 API 使用情况
   - 配置告警
